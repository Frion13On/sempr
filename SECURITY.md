# Реализованные меры безопасности

Документ описывает реализованные механизмы безопасности для курсовой работы.

## 3.5.1 Аутентификация и авторизация пользователей

### Реализовано:

1. **Система ролей пользователей**
   - Три роли: администратор (1), преподаватель (2), студент (3)
   - Роли хранятся в таблице `авторизация` (поле `роль`)
   - Централизованная таблица авторизации с внешними ключами к таблицам ролей

2. **Аутентификация через Flask-Login**
   - Использование `Flask-Login` для управления сессиями пользователей
   - Хранение состояния аутентификации в сессии браузера
   - Автоматическая проверка аутентификации через декоратор `@login_required`
   - Защита от неавторизованного доступа с редиректом на страницу входа

3. **Проверка прав доступа**
   - Функции проверки доступа: `check_admin_access()`, `check_teacher_access()`, `check_student_access()`
   - Декораторы для защиты маршрутов: `require_admin()`, `require_teacher()`, `require_student()`
   - Проверка роли пользователя перед выполнением операций

4. **Хэширование паролей**
   - Использование библиотеки `bcrypt` для хэширования паролей
   - Пароли хранятся в виде хэшей (формат `$2b$...`)
   - Автоматическая миграция старых паролей (plaintext) в хэши при входе
   - Функции: `hash_password()`, `verify_password()` в модуле `password_utils.py`

5. **Валидация входных данных при аутентификации**
   - Проверка наличия логина и пароля
   - Защита от SQL-инъекций через параметризованные запросы

## 3.5.2 Защита данных и шифрование

### Реализовано:

1. **Хэширование паролей (bcrypt)**
   - Алгоритм: bcrypt с 12 раундами (баланс безопасности и производительности)
   - Соль генерируется автоматически для каждого пароля
   - Пароли никогда не хранятся в открытом виде
   - Файл: `password_utils.py`

2. **Безопасное хранение паролей**
   - Пароли хэшируются при создании пользователя (`users_api.py`)
   - Пароли хэшируются при обновлении (`users_api.py`, `account_api.py`)
   - Пароли хэшируются при смене пароля через личный кабинет (`account_api.py`)

3. **Защита секретного ключа**
   - Использование переменных окружения для `SECRET_KEY` (через `.env`)
   - Ключ не хранится в коде, загружается через `python-dotenv`
   - Файл: `app_new.py` (строка 28)

4. **Параметризованные SQL-запросы**
   - Все запросы к БД используют параметризацию (`%s` вместо конкатенации строк)
   - Защита от SQL-инъекций на уровне всех API endpoints
   - Примеры: `auth.py`, `users_api.py`, `account_api.py`, `grades_api.py`

## 3.5.3 Защита от веб-уязвимостей

### Реализовано:

1. **Защита от SQL-инъекций**
   - Все SQL-запросы используют параметризацию
   - Никакая пользовательская строка не подставляется напрямую в SQL
   - Пример: `cursor.execute("SELECT ... WHERE логин = %s", (login,))`
   - Тесты: `test/security/test_security.py::test_login_sql_injection_blocked`

2. **Защита от XSS (Cross-Site Scripting)**
   - Валидация входных данных через функцию `_is_safe_string()` в `users_api.py`
   - Проверка на опасные паттерны: `<script>`, `</script>`, `--`, `;`, `/*`, `*/`, `OR 1=1`
   - Фильтрация всех пользовательских данных перед сохранением в БД
   - Тесты: `test/security/test_security.py::test_add_user_rejects_sql_injection`

3. **Валидация входных данных**
   - Проверка формата email через регулярное выражение
   - Проверка формата телефона (только цифры, до 11 символов)
   - Проверка обязательных полей
   - Проверка, что логин и пароль не совпадают
   - Проверка сложности пароля: не короче 8 символов, наличие строчных и заглавных букв и специального символа
   - Файл: `api/users_api.py` (функция `_validate_and_filter_user_payload`)

4. **Защита от несанкционированного доступа**
   - Все API endpoints защищены декоратором `@login_required`
   - Проверка прав доступа перед выполнением операций
   - Редирект на страницу входа при отсутствии аутентификации

5. **Безопасная работа с сессиями**
   - Использование `SECRET_KEY` для подписи сессий
   - Управление сессиями через Flask-Login
   - Автоматическое удаление пользователя из кэша при выходе

6. **Валидация на клиенте**
   - Проверка данных формы перед отправкой на сервер
   - Валидация email, телефона, проверка совпадения логина и пароля
   - Файл: `static/js/form_utils.js`

## Файлы, связанные с безопасностью

- `auth.py` - аутентификация и проверка прав доступа
- `password_utils.py` - хэширование и проверка паролей
- `api/users_api.py` - валидация и фильтрация пользовательских данных
- `api/account_api.py` - смена паролей с хэшированием
- `app_new.py` - настройка Flask-Login и SECRET_KEY
- `test/security/test_security.py` - тесты безопасности
